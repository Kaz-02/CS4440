02

Hello World in Java

Java 
class Hello {
  public static void main(String[] args) {
    System.out.println("Hello, World!");
  }
}
Rule 01
Must wrap in a class

Java has no standalone functions. Everything lives inside a class. The class name doesn't matter for Hello World.

Rule 02
main() signature is fixed

Must be public static void main(String[] args) exactly — the JVM looks for this specific signature.

Rule 03
static = class-level

static means the method belongs to the class itself, not an instance — so the JVM can call it without creating an object.

Literal examples:

// Integer literals (different bases)
int decimal = 123;
int hex     = 0x10;   // hexadecimal
int octal   = 010;    // octal
int binary  = 0b10;   // binary
long big    = 10l;    // long suffix

// Floating point

float  f = 3.14f;    // float suffix required

// Other
char    c = 'A';
String  s = "hello";
boolean b = true;
Object  o = null;    // any class type can be null
05

Arrays

Declaration, initialization & allocation 
int[] a;                  // declare (preferred style)
int[] b = {1, 2, 3};      // declare + initialize with values
int   c[10];              // ❌ SYNTAX ERROR — size not allowed here
a = new int[5];           // allocate: creates array object of size 5
⚠ Common mistake: You cannot put the array size inside the brackets at declaration time (int c[10] is a compile error). Allocate with new.

Anonymous array — create and pass an array inline without naming it:

Java 
printNumbers( new int[]{1, 2, 3} );
// Similar to "compound literal" in C

Type Inference with var

Since Java 10, you can use var and let the compiler infer the type. Java is still statically typed — the type is fixed at compile time.

Before Java 10 (explicit)

int    x     = 10;
double price = 9.99;
String name  = "John";
User   user  = new User();

Java 10+ (var)

var x     = 10;
var price = 9.99;
var name  = "John";
var user  = new User();
Limitation: var cannot be used where the type cannot be inferred (e.g., uninitialized declarations, method parameters, or return types).
08

Operators & Expressions

Arithmetic
+ - * / % …

Standard math operators. % is remainder.

Assignment
= += -= …

Assignment is also an expression in Java (has a value).

Comparison
== != > >= …

Must produce a true boolean — no truthy/falsy like Python or C.

Increment
++ --

Pre (++x) increments before use; post (x++) increments after.

Logical
&& || !

Short-circuit evaluation: && stops at first false, || at first true.

Conditional
condition ? a : b

Ternary operator — compact inline if/else expression.

⚠ Java gotcha: Boolean expressions must be actual booleans. Unlike Python (if myList:) or C (if (n)), Java will not accept integers or objects as conditions.
09

Statements

Assignment, branching (if/else, switch), and loops (while, for) work like C. The notable addition is the for-each loop:

Java — for-each 
int[] numbers = {1, 2, 3, 4, 5};

for (int num : numbers) {         // for each num in numbers
  System.out.println(num);
}
The for-each loop works on any array or class implementing Iterable (like ArrayList, Set, etc.).

Classes & Methods

Java — Dog class

public class Dog {
  private String name;
  private int age;

  public Dog(String name, int age) {
    this.name = name;
    this.age  = age;
  }

  public void sit() {
    System.out.println(
      name + " is now sitting.");
  }

  public String getName() {
    return name;
  }
  public void setName(String name) {
    this.name = name;
  }
}

var monty = new Dog("Monty", 2);
monty.sit();

Python — equivalent

class Dog:
  def __init__(self, name, age):
    self.name = name
    self.age  = age

  def sit(self):
    print(
      f"{self.name} is now sitting.")


monty = Dog("Monty", 2)
monty.sit()

Key rules about Java classes & methods:

File rule
One public class per file

A public class must be in a file with the exact same name, e.g. Dog.java.

Constructor
Same name, no return type

Constructor name matches the class and has no return type — not even void.

Methods
No defaults, supports overloading

No default parameter values or keyword arguments, but multiple methods can share a name with different parameter types.

Convention
Private fields + getters/setters

Fields are typically private; access is provided through public getter/setter methods.

11

Properties (Getters & Setters)

Java defines a "property" by convention — a private field exposed through getter and/or setter methods. The naming convention is critical:

Read-only
getter only

Only a getXxx() method. External code can read but not write.

Write-only
setter only

Only a setXxx() method. Rarely used.

Read-write
both

Both getter and setter. The standard case.

Boolean
isXxx()

Boolean properties use isXxx() instead of getXxx().

Java — User class properties 
class User {
  private String firstName;   // property: firstName (read-write)
  private String lastName;    // property: lastName  (read-write)

  public String getFirstName() { return firstName; }
  public String getLastName()  { return lastName;  }
  public String getName()      { return firstName + " " + lastName; }
  // "name" is a computed read-only property (no setter)
}
Pro tip: Let your IDE (IntelliJ, Eclipse, VS Code) generate getters and setters automatically — it avoids typos and saves time.

Packages & Imports

Java classes should live inside packages. Packages use the reverse URL naming convention — from general to specific:

URL vs Package naming 
// URL:     cs.calstatela.edu        (specific → general)
// Package: edu.calstatela.cs.cs3035.animal  (general → specific)

package edu.calstatela.cs.cs3035.animal;

public class Dog {
  private String name;
  private int age;
  …
}

Package structure must mirror directory structure:

src/
  edu/
    calstatela/
      cs/
        cs3035/
          animal/
            Dog.java

Importing classes from packages:

import edu.calstatela.cs.cs3035.animal.Dog; Import specific class
import edu.calstatela.cs.cs3035.animal.*; Import everything (not sub-packages)
edu.calstatela.cs.cs3035.animal.Dog myDog = … No import — use fully qualified name
Note: import * imports all classes in a package but not its sub-packages. Sub-packages must be imported separately.
