Names and Scopes

Study notes compiled from lecture slides

01

Names & Binding

A name is any identifier used to refer to an entity in a program — a variable, function, class, constant, etc.

Binding is the act of associating a name with a specific entity. The same name can be bound to different entities in different contexts.

from foo import *   # imports add() function and PI constant

print(add(2, 3))     # 'add' is bound to the imported function
print(PI)

add = 100              # 'add' is NOW rebound to the integer 100 !
print(add(2, 3))     # ❌ TypeError — add is no longer a function
Reusing a name to refer to different entities in the same scope is generally a bad programming practice. It causes confusion and hard-to-find bugs.
02

Scope

A scope is the region of a program where a particular name binding is valid. Restricting scope prevents name conflicts between different parts of a program.

C — two separate scopes for x

void foo() {
    int x;  // x is int, in foo's scope
}

void bar() {
    double x;  // x is double, in bar's scope
}

Key insight

Both functions define a variable named x, but they live in separate scopes — there is no conflict.

Without scoping, every name in a program would need to be globally unique. That would be unworkable.

03

Blocks

A block is a group of statements treated as a single unit. Blocks usually introduce a new scope. There are two types:

Block TypeDescriptionExample
Function blockThe body of a function or methodvoid foo() { … }
Inline blockA block inside a function — loop body, if body, etc.for (…) { … }
LanguageBlock Syntax
C / Java / JavaScript{ … } — curly braces
PythonIndentation level
Ada / Pascalbegin … end keywords

Java — nested blocks create nested scopes

public void checkNumbers(int[] numbers) {         // function scope
    for (int i = 0; i < numbers.length; i++) {    // for-block scope
        int currentValue = numbers[i];
        if (currentValue > 100) {                   // if-block scope
            String message = "Large number!";
            System.out.println(message);
        }
        System.out.println(message);  // ❌ Error — message is out of scope
    }
    System.out.println(currentValue); // ❌ Error — currentValue is out of scope
}
Function scope — checkNumbers()
for-block scope

int currentValue — visible here and in nested blocks

if-block scope

String message — only visible inside this if-block

message here ❌ — if-block already ended

currentValue here ❌ — for-block already ended

Python's inline blocks (like for or if) do not create new scopes — variables defined inside them are visible in the enclosing function. This is a key difference from C/Java.
04

Point of Declaration / Definition

When does a variable's scope actually start? It depends on the language:

C-style — scope starts at declaration

int foo() {
    print(x);    // ❌ x not yet declared
    int x = 3;
}

x's scope starts at the int x = 3; line.

Python — scope starts at function start

def foo():
    print(x)  # ❌ UnboundLocalError
    x = 3

Python sees that x is assigned in foo, so it treats x as local for the whole function — but it's unbound at the print line.

❓ Why does the Python version also error, if scope starts at function start?
Because Python hoists the name (marks x as local to foo) but does not hoist the value. So x exists as a local name, but accessing it before the assignment gives an UnboundLocalError.

The same issue arises with mutually recursive functions in C — since foo calls bar which isn't declared yet at that point. C requires a forward declaration; Python doesn't:

C — needs forward declaration

int foo(int x) {
    int bar(int);  // forward decl ✅
    return bar(x + 1);
}
int bar(int x) {
    return foo(x - 1);
}

Python — no issue (functions resolved at call time)

def foo(x):
    return bar(x + 1)  # ✅ ok

def bar(x):
    return foo(x - 1)
05

Static Scope vs. Dynamic Scope

Consider this code — what does it print?

x = 10
function showX():
    print x
function parent():
    x = 20
    showX()
parent()
🗂 Static (Lexical) Scope

Scope determined by physical location in source code.

Determined at compile time.

showX() looks up x in the scope where it was written — which is the global scope → finds x = 10.

Output: 10

Used by: Python, Java, C, JavaScript, most modern languages

📞 Dynamic Scope

Scope determined by execution call stack at runtime.

Determined at runtime.

showX() looks up x in the calling frame first → finds x = 20 in parent().

Output: 20

Used by: Emacs Lisp, some shells (Bash), early Lisps

Static scope — name lookup follows source code structure

Global x = 10
showX() print x ← looks up x in its lexical parent → global → finds 10
parent() x = 20   showX() ← x=20 is local to parent, not visible to showX
06

Frames & the Call Stack

A frame (a.k.a. stack frame or activation record) is a data structure pushed onto the call stack each time a function is called. It holds everything that function needs to execute.

Parameters
arguments passed to the function
Local Variables
variables declared inside the function
Return Address
where to resume execution after return
Saved Registers
CPU state to restore on return

During the execution of parent() which calls showX(), the call stack looks like:

Call Stack(grows upward)
global x = 10
parent() x = 20
showX() (no x here) TOP

Under dynamic scope, showX() searches up the call stack frames and finds x = 20 in parent(). Under static scope, it ignores the stack and follows the lexical structure instead.

Frames are destroyed (popped off the stack) when the function returns — this is exactly why local variables cease to exist after a function call ends (automatic storage).
07

Nested Functions & Closures

Python (and many languages) allows defining a function inside another function. The inner function can access variables from its enclosing scope — this is called a closure.

x = 0

def foo():
    x = 2
    def baz():           # inner function — defined inside foo
        print(x)         # captures x from foo's scope
    return baz            # returns the function object itself

def bar():
    x = 1
    foo()()              # foo() returns baz; () calls it

bar()
2
❓ Why does it print 2 and not 1 or 0?
Static (lexical) scope. baz was defined inside foo, so it captures foo's x = 2. Even though bar calls it (and bar has its own x = 1), that doesn't matter — scope follows source code structure, not the call chain. The returned baz function is a closure that "closes over" foo's x.
08

Overloading & Shadowing

Overloading

Same name refers to multiple entities in the same scope.

Extra info (e.g. argument types) is needed to tell them apart.

Example: two methods named display() — one takes no args, one takes a String.

Shadowing

An inner variable has the same name as an outer variable.

The inner variable hides the outer one within its scope.

Example: parameter name shadows the field this.name.

Java — overloading + shadowing in the same class

public class Foo {
    String name = "John";          // class field

    void display() {               // overload 1 — no parameter
        System.out.println("Hello " + name);  // uses field 'name'
    }

    void display(String name) {    // overload 2 — String parameter
        // parameter 'name' SHADOWS the field 'name'
        System.out.println("Hello " + name);  // uses parameter, not field
        // use 'this.name' to access the field explicitly
    }
}
To access a shadowed outer variable in Java, use this.name. In Python, use global x or nonlocal x to write to an outer variable instead of creating a new local one.
09

Python: global & nonlocal

In Python, assignment always creates a new local variable in the current scope. To modify a variable in an outer scope, you must declare it explicitly.

Without global/nonlocal

x = 10

def foo():
    x = 20           # new local x
    def bar():
        x = 30       # new local x
    bar()
    print(x)         # prints 20

foo()
print(x)            # prints 10
20 10

With global / nonlocal

x = 10

def foo():
    x = 20
    def bar():
        nonlocal x   # modify foo's x
        x = 30
    bar()
    print(x)         # prints 30

foo()
print(x)            # prints 10
30 10
KeywordEffectUse case
global xRefers to the module-level (global) xModifying a global variable from inside any function
nonlocal xRefers to the nearest enclosing (non-global) xModifying a variable in an outer (but not global) function
